Drive to Explorer

Google Drive (Web) で開いているフォルダを、ワンクリックで Windows エクスプローラーに。

v0.3.0 Chrome / Edge / Brave / Vivaldi Windows 10 / 11 UI からワンクリック更新

このツールは何をする？

Drive Web で表示中のフォルダを、Drive for desktop がミラーしているローカルフォルダとして エクスプローラーで即座に開きます。逆方向（エクスプローラー → Drive Web）にも対応。

前提: ① Drive for desktop がドライブレター（例 I:\）でマウント済み。 ② Chromium 系ブラウザ（Chrome / Edge / Brave / Vivaldi）。配布 zip には Native Host の .exe が同梱されているため Python は不要です。

SETUPはじめての導入（4 ステップ）

配布された zip を、バージョン番号を含まない安定フォルダ（例 F:\Tools\drive-to-explorer\）に解凍してから始めてください。

1

拡張機能を読み込む

ブラウザの拡張機能ページ（chrome://extensions / edge://extensions など）を開き、右上の デベロッパーモードを ON → 「パッケージ化されていない拡張機能を読み込む」 で、解凍した extension/ フォルダを選択します。

読み込むと拡張機能 ID は次の値に固定されます（全 PC 共通）:

pkiecgchhhhcgnjjgofmlfobbhacdcge
2

Native Host を登録する

native-host\install.bat をダブルクリック。Extension ID を聞かれたら、そのまま Enter（上の ID が既定値で入っています）。各ブラウザのレジストリに自動登録されます。

  • 登録後は ブラウザを完全終了 → 再起動（タスクマネージャーでプロセスが残っていないか確認）。
3

ローカルルートパスを設定する

拡張アイコンを右クリック →「オプション」を開き、ローカルルートパスに Drive for desktop のドライブレター（例 I:\）を入力 →「ルート存在チェック」→「保存」。

共有ドライブ / マイドライブの配下は自動で順次試行するので、ルートはドライブレターだけで OK です。

4

Google にサインインする（推奨）

オプション画面の Drive REST API (OAuth) セクションで 「サインイン」 をクリック → アカウント選択 →「許可」。配布版は既定 Client ID 同梱なので、これだけで API モードが有効になります。

「このアプリは確認されていません」と出たら、「詳細」→「Drive to Explorer に移動」で進みます。自作アプリの標準警告で問題ありません。
使い方: Drive Web でフォルダを開き、右クリック →「フォルダ情報」→「エクスプローラーで開く」。 またはツールバーの拡張アイコンから。逆方向はエクスプローラーでフォルダを右クリック →「Drive で開く」（shell-integration\install_shell.bat 実行後）。

UPDATEアップデートは UI からワンクリック

毎回のインストールやコピペは不要です。オプション画面の 「今すぐ更新」 ボタンを押すだけで、 ダウンロード → 適用 → 拡張の自動再読み込みまで完結します。

① 「今すぐ更新」を押す
オプション画面ヘッダーのボタン。確認ダイアログで OK。
↓
② 自動でダウンロード＆入れ替え
最新版を取得し、ファイルを上書き。進捗が画面に表示されます。
↓
③ 自動で再読み込み
完了すると拡張が自動でリロードされ、新バージョンが有効に。ブラウザ再起動も bat 実行も不要。
ボタンを押す前に、開いている Drive タブを再読み込みしておくとより安全です。 まれに更新が失敗した場合のフォールバックとして、インストール先の update.bat をダブルクリックする方法も残してあります。
仕組み: ブラウザ拡張は自分のファイルを書き換えられないため、同梱の Native Host（.exe）が GitHub Releases から最新版を取得・展開し、ファイルを入れ替えます。拡張側は完了を検知して自動リロードします。

PRIVACY権限とプライバシー

安心して使えるよう、要求する権限は最小限に絞っています。

🔒
OAuth スコープは drive.metadata.readonly のみ。
フォルダ名と親フォルダ情報だけを参照します。ファイルの中身は一切読み込みません。ダウンロードも書き込みも行いません。
サインインは任意です。未サインインでも Drive 画面の解析（DOM 解析モード）で動作しますが、 深い階層や共有ドライブの正確な解決には API モード（サインイン）を推奨します。

HELP困ったとき

配布先で自己解決できるよう、よくある症状と対処をまとめました。

「Native host has exited」「host not found」と出る
Native Host が未登録か、ID 不一致です。
  • native-host\install.bat を再実行（ID を聞かれたら Enter）。
  • 実行後、ブラウザを完全終了 → 再起動。
「今すぐ更新」が失敗する / 進まない
  • ネットワーク（GitHub への接続）を確認。
  • 開いている Drive タブを再読み込みしてから再試行。
  • それでもダメなら、インストール先の update.bat をダブルクリック（フォールバック）。
「ローカルパスが見つかりません」と出る
該当フォルダが Drive for desktop にミラーされているか確認してください。 popup や通知に「試したパス」が表示されます。共有ドライブは <root>\共有ドライブ\<ドライブ名>\... 配下にミラーされます。 取りこぼす場合は右クリックメニュー経由をお試しください。
何も起きない / 動作がおかしい
オプション画面の「デバッグ」を ON にすると、ブラウザの開発者ツールに [DTE] ログが出ます。 Native Host のログは %TEMP%\drive_to_explorer_host.log に記録されます。
アンインストールしたい
① ブラウザの拡張機能ページから「Drive to Explorer」を削除。 ② native-host\uninstall.bat を実行（レジストリから削除）。 ③ 右クリックメニューを入れた場合は shell-integration\uninstall_shell.bat も実行。
