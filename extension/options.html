Drive to Explorer

「今すぐ更新」で最新版を自動ダウンロード→適用→拡張の自動再読み込みまで行います （ブラウザ再起動・コピペ・bat 実行は不要）。

ローカルルートパス

Drive for desktop がマウントされているドライブを指定します。複数行に分けると上から順に試行します。

例: M:\ や I:\。Drive 階層（マイドライブ / 共有ドライブ / フォルダ名…）が このルート配下にミラーされている前提です。

Native Host インストール補助

この拡張機能 ID を native-host\install.bat 起動時に貼り付けてください。 install.bat が manifest.json を更新し、各ブラウザのレジストリに登録します。

Drive REST API (OAuth)

API 経由で Drive 階層を解決します（DOM 解析より高信頼）。配布版には既定の OAuth Client ID が同梱されているので、 通常は 「サインイン」を押すだけで利用開始できます。 独自の Client ID を使う場合のみ下欄に入力してください。

権限スコープは drive.metadata.readonly のみ。 フォルダ名・親フォルダ情報だけを参照し、ファイルの中身は一切読み込みません。

API モード診断

アクティブな Drive タブの URL から folder/file ID を抽出し、API でフォルダ階層を解決できるかテストします。

デバッグ

Drive ページ / Service Worker のコンソールに [DTE] ログを大量出力します。 通常はオフのままで構いません。
