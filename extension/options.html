Drive to Explorer - 設定

Drive for desktop がマウントされているローカルのルートパスを指定します。複数指定する場合は 1 行に 1 つずつ。

例: M:\ や G:\。Drive 階層 (マイドライブ / 共有ドライブ / フォルダ名…) が、このルート配下にそのままミラーされていることが前提です。
複数指定すると、上から順に試行して最初に存在するパスを使います。

1 行に 1 つ

Native Host インストール補助

この拡張機能 ID を native-host\install.bat 起動時に貼り付けてください。
install.bat が manifest.json を自動更新し、各ブラウザのレジストリに登録します。

Drive REST API (OAuth)

本拡張は API 経由で Drive 階層を解決します（DOM 解析より高信頼）。
配布版には既定の OAuth Client ID が同梱されているので、 通常は 「サインイン」ボタンを押すだけで利用開始できます。
独自の Client ID を使いたい場合のみ、下の欄に入力して「Client ID 保存」してください。

これを Google Cloud の OAuth クライアントに「承認済みリダイレクト URI」として登録
「ウェブ アプリケーション」型のクライアント ID
API モード診断

アクティブな Drive タブの URL から folder/file ID を抽出して、Drive REST API でフォルダ階層を解決できるかテストします。

デバッグ

Drive ページ / Service Worker のコンソールに [DTE] ログを大量に出力します。 通常の利用ではオフのままで構いません。
